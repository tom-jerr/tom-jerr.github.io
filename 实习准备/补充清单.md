基于当前笔记分析，以下是需要补充的知识点。

## 🔴 高优先级（面试高频）

### CUDA / Kernel

- [ ] **双缓冲 + cp.async 流水线**
  - CUTLASS 3.x MainLoop Pipelining
  - 三级流水线设计（load-compute-store）
  - `mbarrier` 使用方式
  - 参考：https://github.com/NVIDIA/cutlass/blob/main/media/docs/pipeline.md

- [ ] **Swizzling 实现细节**
  - XOR swizzling 公式推导
  - `Swizzle<B,M,S>` 参数含义
  - 与 padding 的对比实验

- [ ] **wgmma 指令（Hopper）**
  - 与 wmma 的区别
  - 异步执行模型
  - 与 TMA 配合的 tile 加载

- [ ] **NCU SOL 分析**
  - Speed of Light 百分比解读
  - Roofline Model 画法
  - memory-bound vs compute-bound 判断

### MoE / EP

- [ ] **All-to-All 通信**
  - 通信量公式：`2 * (n-1) / n * data_size`
  - 跨节点扩展性问题
  - 与 NVSwitch 拓扑的关系

- [ ] **Expert Capacity 机制**
  - capacity factor 设置策略
  - token dropping 的影响
  - auxiliary load balancing loss

- [ ] **DeepSeek-V3 MLA**
  - Multi-head Latent Attention 原理
  - KV cache 压缩比
  - 与 GQA 的对比

### Profiler

- [ ] **NCU 指标解读**
  - Achieved vs Theoretical Occupancy
  - L1/L2 Hit Rate 含义
  - Memory Throughput SOL
  - Warp Stall 分类饼图

- [ ] **Nsys 分析流程**
  - CPU-GPU gap 识别
  - Kernel launch overhead 定位
  - CUDA Graph 前后对比

### 分布式

- [ ] **通信原语算法**
  - Ring All-Reduce 时间公式：`2*(n-1)/n * M/B`
  - Tree vs Ring 选择
  - NCCL 调优参数

- [ ] **TP/PP 混合选择**
  - TP8 vs TP4+PP2 权衡
  - 跨节点 TP 的 IB 带宽需求
  - Pipeline Bubble 估算

- [ ] **vLLM 架构**

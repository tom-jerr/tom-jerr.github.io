C Prime Plus 复习

单词类型

  • 标识符，运算符，分隔符，常量，注释符，关键字
  • 关键字：auto, extern, register, static

C 预处理器指令

  • #inlcude 在编译前对源代码进行一些预处理
  • #include <头文件>会将头文件的内容在引用该头文件的源文件中展开；避免重复引用，头文件要条件编译或#pragma once

数据类型

整数溢出

  • 如果表示的值超过相应的取值范围，会重新从最小值开始计算
  • 溢出是 C 语言未定义行为，结果取决于编译器

char 类型

  • 存储字符，但从技术上看，char 类型存储的实际上是整数不是字符
  • 表示范围取决于编译器；可能是有符号，也可能是无符号

_Bool 类型

  • C99 标准添加了_Bool 类型；用于表示布尔值；实际上也是整数类型

浮点数

  • 上溢：赋一个无穷大的特定值
  • 下溢：计算过程中损失了原来末尾有效位上的数字
  • 特殊值：NAN

复数和虚数

  • 两个值，分别表示实部和虚部：float _Complex, double _Complex, long double _Complex
  • 表示虚部：float _Imaginary, double _Imaginary, long double _Imaginary

刷新输出 (console)

  • 控制台是行缓冲，最后打印换行符就能打印整行数据
  • fflush 刷新缓冲区，打印字符

字符串

  • char 数组存放字符串；字符数组末位加’\0’
  • sizeof 计算末尾空字符；strlen 不计算末尾空字符
  • 存放在常量区，.rodata 段，一般不能修改

printf()

  • “%-20s”：从字段的左侧打印该项
  • “%#o”：把结果转换为另一种形式
  • “%010d”：对于数值格式，用前导 0 代替空格填充字段宽度

运算符

  • sizeof()返回 size_t 类型的值，是一个无符号整数类型

  • 递增和递减运算符的优先级只比()小；只能影响一个变量；(x*n)++是无效的

  • 在函数中出现多个相同变量，使用递增或递减；可能出现错误，因为编译器执行的顺序未知

    • 一个变量出现在一个函数的多个参数中，不要对该变量使用递增或递减
    • 一个变量多次出现在一个表达式中，不要对该变量使用递增或递减

  • C 并未指明在子表达式求值后递增还是整个表达式求值后递增

优先级

  • +，-，++，–，sizeof 是从右向左结合
  • =是从右向左结合
  • .和->还有[]优先级比*高；++，–优先级比*高

分支和跳转

  • 为了提高效率，应该将情况大概率发生的语句放在 if 中
    • 因为现代 CPU 流水线结构存在分支错误惩罚，其代价接近 19 个时钟周期
  • switch 语句比 if-else 执行效率更高
    • 编译时将跳转地址存储在.rodata 中
    • 符合条件后直接跳转到该指令处，不再继续执行 if 判断
    • switch 不使用浮点数判断；因为浮点数在计算机中的表示并不精确；不能被转化为正确的二进制数
    • switch 不支持字符串作为比较条件，该语句被设计为处理数值或特定的枚举类型

IO

  • 字符输出存在缓冲区
    • 行缓冲：控制台输出，输入换行符后，打印缓冲区的内容
    • 全缓冲：缓冲区全满后，才将缓冲区的所有内容输出

重定向

  • 实际上是将关闭待执行命令的 stdin，将其替换成输入文件的 fd；或者是关闭待执行命令的 stdout，将其替换成输出文件的 fd

函数

尾递归

  • 递归形式与函数定义相同；编译器会将其优化成循环结构

数组和指针

指定初始化器

  • 数组初始化可以用花括号，但是赋值不行

变长数组 VLA

  • 普通数组是静态分配内存；但是 VLA 可以动态分配内存

复合字面量 (C99)

伸缩型数组成员 (C99)

  • 该数组不会立即存在
  • 在为结构体动态分配内存时分配
    规则
  • 必须是结构体最后一个成员
  • 结构中必须有一个以上的成员
  • 声明类似普通数组, 只是方括号是空的

union

  • 联合结构, 在同一个内存空间中存储不同的数据类型 (不是同时存储)
  • 联合结构中只能定义一个值

enum

  • 枚举类型
  • 从技术上来看, 本身是 int 类型的常量

字符串函数

  • 用字符串常量初始化 char 数组；该数组得到的是字符串常量的副本

查找

  • 在一个字符串中查找一个特定的字符
    • strchr 查找第一个匹配的位置
    • strrchr 查找最后一个匹配的位置
  • 查找一组字符中任意一个出现的第一个位置
  • 查找一个子串
  • 查找一个字符串前缀
    • group 指定一个或多个字符，返回 str 起始部分匹配的任意字符的字符数
    • group 指定一个或多个字符，返回 str 起始部分不与 group 匹配的任意字符的字符数
  • 查找标记：隔离各个单独的部分（token 标记），丢弃分隔符
  • 错误信息：将 OS 设置的 errno 返回（全局的变量）

存储类别和链接

  • 编译单元：一个源文件以及它所包含的头文件为一个编译单元

存储期

  • 静态存储期：程序执行期间一直在
  • 自动变量有自动存储期，块作用域，无链接
  • 寄存器变量是向系统请求使用寄存器来存变量；有自动存储期，块作用域，无链接
  • 多文件下，变量只有一个定义；其他文件引用该变量声明时，使用 extern
  • 全局变量是外部链接的静态变量
  • 内部链接的静态变量：只能用于本文件
  • 静态存储类别所用的内存数量在编译时确定存在全局（静态）存储区；
  • 自动变量一般存在栈上

ANSI C 类型限定符

  • volatile：告知编译器每次取变量从内存中取，用于多线程共享
  • restrict：只用于指针，表明该指针式访问数据对象的唯一且初始的方式；即两个指针不会指向同一块内存，编译器可以优化程序

文件 I/O

getc() & putc()

  • 从指定的流 stream 获取下一个字符（一个无符号字符），并把位置标识符往前移动。
  • 从标准输入 stdin 获取一个字符（一个无符号字符）。这等同于 getc 带有 stdin 作为参数。
  • 把参数 char 指定的字符（一个无符号字符）写入到指定的流 stream 中，并把位置标识符往前移动。

fseek() & ftell()

  • 设置流 stream 的文件位置为给定的偏移 offset，参数 offset 意味着从给定的 whence 位置查找的字节数。
  • 如果成功，则该函数返回零，否则返回非零值。

参数

  • stream – 这是指向 FILE 对象的指针，该 FILE 对象标识了流。
  • offset – 这是相对 whence 的偏移量，以字节为单位。
  • whence – 这是表示开始添加偏移 offset 的位置。它一般指定为下列常量之一：
常量 描述
SEEK_CUR 文件指针的当前位置
SEEK_END 文件的末尾
SEEK_SET 文件的开头
  • 返回给定流 stream 的当前文件位置。
  • 距离文件开始处的字节数

ungetc()

  • 把字符 char（一个无符号字符）推入到指定的流 stream 中，以便它是下一个被读取到的字符。
  • 如果成功，则返回被推入的字符，否则返回 EOF，且流 stream 保持不变。

fflush()

  • 刷新流 stream 的输出缓冲区。（将所有未写入数据发送到指定文件中）
  • 如果成功，该函数返回零值。如果发生错误，则返回 EOF，且设置错误标识符（即 feof）。

setvbuf()

  • 定义流 stream 应如何缓冲

参数

  • stream – 这是指向 FILE 对象的指针，该 FILE 对象标识了一个打开的流。
  • buffer – 这是分配给用户的缓冲。如果设置为 NULL，该函数会自动分配一个指定大小的缓冲。
  • mode – 这指定了文件缓冲的模式：
模式 描述
_IOFBF 全缓冲：对于输出，数据在缓冲填满时被一次性写入。对于输入，缓冲会在请求输入且缓冲为空时被填充。
_IOLBF 行缓冲：对于输出，数据在遇到换行符或者在缓冲填满时被写入，具体视情况而定。对于输入，缓冲会在请求输入且缓冲为空时被填充，直到遇到下一个换行符。
_IONBF 无缓冲：不使用缓冲。每个 I/O 操作都被即时写入。buffer 和 size 参数被忽略。
  • size –这是缓冲的大小，以字节为单位。

fread() & fwrite()

  • 从给定流 stream 读取数据到 ptr 所指向的数组中。

参数

  • ptr – 这是指向带有最小尺寸 size*nmemb 字节的内存块的指针。
  • size – 这是要读取的每个元素的大小，以字节为单位。
  • nmemb – 这是元素的个数，每个元素的大小为 size 字节。
  • stream – 这是指向 FILE 对象的指针，该 FILE 对象指定了一个输入流。

返回值

  • 成功读取的元素总数会以 size_t 对象返回，size_t 对象是一个整型数据类型。如果总数与 nmemb 参数不同，则可能发生了一个错误或者到达了文件末尾。

把 ptr 所指向的数组中的数据写入到给定流 stream 中。

结构体分配内存

  • 结构体的首址一定是第一个数据类型的地址
  • 依据结构体中内存最大的数据和#pragma pack(n)中的 n 二者取最小值最为内存对齐的粒度
  • 最终结构体的大小是粒度的整数倍

C 预处理器

可变参数列表

  • stdarg 宏
  • 可变参数必须从头到尾按顺序访问

退出函数

abort()

  • 中止程序执行，直接从调用的地方跳出。

atexit()

  • 当程序正常终止时，调用指定的函数 func。您可以在任何地方注册你的终止函数，但它会在程序终止的时候被调用。

exit()

  • 立即终止调用进程。任何属于该进程的打开的文件描述符都会被关闭，该进程的子进程由进程 1 继承，初始化，且会向父进程发送一个 SIGCHLD 信号。

链接

本文最后更新于：2023年7月4日 下午

前言

Linux多线程

  • fork：复制当前进程（产生新任务）
  • clone：创建子进程并从指定位置开始执行（产生新线程）

可重入

  • 函数没有执行完成，由于外部因素或者自身调用再次进入该函数执行

  • volatile阻止编译器将数据存放在寄存器，造成过度优化

链接

编译

  • 每个编译单元只能有一个强符号，但是可以有多个弱符号

  • readelf观察符号表

两步链接

  • 空间与地址分配，设置全局符号表，将目标文件的各个段进行合并，计算合并后的段长度与位置
  • 符号解析与重定位
    • （链接时）重定位：存在重定位表，包含重定位入口偏移和重定位入口的类型和信息

全局构造

  • Linux下程序入口地址：_start
  • ELF中的段
    • .init：main调用前，Glibc初始化部分
    • .fini：保存终止代码指令，main正常退出后执行

静态链接库

  • 一个目标文件只有一个函数
  • 避免空间浪费

可执行文件的装载

  • 操作系统将进程虚拟地址空间划分成一个个VMA；将相同权限的文件映射到同一个VMA中
    • 代码VMA
    • 数据VMA
    • 堆VMA
    • 栈VMA

动态链接

  • 静态链接下，整个程序只有一个ELF文件，不可分割
  • 动态链接下，一个程序被分成了若干个文件
  • 共享对象最终装载地址不确定的（.so文件）

装载时重定位

  • PLT延迟绑定：函数第一次用到时才进行绑定

特点

  • 多个程序虽然可以共享同一个动态链接库中的相同代码，但每一个进程都为动态链接库使用的全部数据分配了自己的地址空间。而共享内存是代码和数据均被多个程序所共享。
  • 动态链接库只是实现了代码的复用，对于数据，各个程序要自己保存。
  • 共享内存用于进程间通信，主要用于数据的内存共享。这也从侧面反映出动态链接库并不能用于实现进程之间通信。

内存

栈

  • esp始终指向栈的顶部，ebp被称为帧指针

堆

  • malloc中申请的空间在虚拟地址空间是连续的，但物理空间不一定

堆分配算法

  • 空闲链表组织

  • 位图（Bitmap）

运行时库

linux入口函数

  • 在main函数调用前调用
  • glibc的全局构造函数放置在.ctors段中；手动在该段中添加函数指针，让函数在main函数调用前被调用
  • 可以使用__attribute__((constructor))实现

中断切换堆栈

  • int 0x80进行中断；寻找中断向量表对应位置
  • 在内核中压入用户态的SS、ESP、EFLAGS、CS、EIP

传参

  • asmlinkage：是一个宏，强迫函数在栈上取参数

write系统调用

  • 汇编形式，可以用gcc内联汇编改写（-m32：64位系统运行32位汇编）
  • 系统调用统一使用int $0x80；跳转到中断向量表中按syscall number寻找对应的中断程序入口地址

C&C++运行时库的实现

堆的实现

  • Linux下使用brk将数据段结束地址向后调整32MB
  • 仅仅只是分配了对应的虚拟地址空间，只有进程要访问才会分配地址空间
  • “m”约束表示直接在内存上操作数据，而不是传入寄存器后再操作

编译库

  • 静态库：rs -rs *.o

  • 动态库：gcc test.c -fPIC -shared -o libtest.so

new&delete

  • 反汇编后出现_Znwj
  • 使用反修饰符（c++filt _Znwj），是operator new函数

全局构造和析构函数

  • Glibc需要两个目标文件实现全局构造和析构

atexit函数

  • 所有全局对象的析构函数都是通过atexit或其他类似函数注册的，实现程序退出时执行的目的

main参数

  • exit实现是使用Linux的系统调用（1号）

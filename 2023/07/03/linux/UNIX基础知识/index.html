UNIX基础

本文最后更新于：2023年7月4日 下午

前言

基础知识

部分结构

DIR

  • 结构体类似FILE，是一个内部结构

  • DIR *opendir(const char *pathname)，即打开文件目录，返回的就是指向DIR结构体的指针，而该指针由以下几个函数使用:

dirent

  • 目录文件结构体，directory file
  • 这种文件包含了其他文件的名字以及指向与这些文件有关的信息的指针
  • 主要起索引的作用

  • 通过readdir函数读取到的文件名存储在结构体dirent的d_name成员中

  • int stat(const char *file_name, struct stat *buf)的作用就是获取文件名为d_name的文件的详细信息，存储在stat结构体中。

stat

file结构体

  • “struct file结构体定义在include/linux/fs.h中定义。文件结构体代表一个打开的文件，系统中的每个打开的文件在内核空间都有一个关联的 struct file。它由内核在打开文件时创建，并传递给在文件上进行操作的任何函数。在文件的所有实例都关闭后，内核释放这个数据结构。在内核创建和驱动源码中，struct file的指针通常被命名为file或filp。”

  • FILE结构是在内核打开文件时创建的。

获取目录下详细文件信息

  • 首先，我们使用opendir函数打开目录a，返回指向目录a的DIR结构体c。
  • 接着，我们调用readdir( c)函数读取目录a下所有文件（包括目录），返回指向目录a下所有文件的dirent结构体d。
  • 然后，我们遍历d，调用stat（d->name,stat *e）来获取每个文件的详细信息，存储在stat结构体e中。

chdir()

  • chdir（）用户将当前的工作目录改变成以参数路径所指的目录

输入和输出

  • open, read, write, lseek, close都提供了不带缓冲的IO；都使用文件描述符fd

出错处理

  • 多线程存取errno，局部errno

打印出错信息

用户

  • 用户ID：使用32位整数

  • 组ID：将组名映射成组ID，使用32位整数，/etc/group

  • 附属组ID：/etc/group

信号：SIGINT

处理方式：

  • 忽略信号
  • 按系统默认方式处理。对于除数为0，系统默认方式是终止该进程
  • 提供一个函数，信号发生时调用该函数（捕捉信号）

中断键：Ctrl+c

退出键：Ctrl+\

  • 可以调用kill函数实现

UNIX标准

函数

  • 以__SC开头的是系统限制（sysconf）
  • 以__PC开头的是运行时限制（pathconf、fpathconf）

不确定的运行限制

  • 路径名：PATH_MAX：使用pathconf获取结果后+1，包括最后的Null
  • 最大打开文件数：OPEN_MAX：dup函数在打开文件超过OPEN_MAX会返回一个特殊值

更改运行限制

  • 在 sysconf 没有能够准确获得 OPEN_MAX 限制的时候，可以使用 getrlimit 来获得。
  • 如果是无限大，赋一个初始值；如果没有则报错

基本系统数据类型

  • clock_t：时钟滴答计数器
  • comp_t：压缩的时钟滴答
  • dev_t：设备号
  • ptrdiff_t：两个指针相减结果（带符号）
  • …

功能测试宏

  • defined (_POSIX_C_SOURCE)

习题

  • const向左结合

  • int* const：指向Int的常量指针

  • const int*：指向Int常量的指针

perror为什么传入常量

  • 如果perror会修改*s，则会修改s指向的字符串的内容。因此，您需要perror(const char *s)（指向常量值的指针）以确保您的字符串未被perror修改。
  • 而strerror只将errno的值传入，映射成一个字符串，不会对原值产生影 响

内部链接性

语言链接性

存储方案和动态分配

  • 常规new运算符：只有一个参数表示大小

  • 定位new运算符：有两个参数，将其中一个地址参数作为存放new出新对象的地址

    • 不能替换，可以重载
    • 如果分配的是静态内存，不能使用delete进行内存释放
    • delete只能释放在堆上new出的新内存

继承与派生

继承

  • 派生类继承基类的公有接口和数据成员
  • 基类需要实现虚析构函数：保证多态情况下子类内存的正确释放
  • 友元函数不能继承；可以通过强制类型转换，调用基类的友元函数

虚拟功能

  • 虚函数是基类中的函数，使用关键字 virtual 声明。在基类中定义一个虚拟函数，在派生类中定义另一个版本，向编译器发出信号，表明我们不希望此函数的静态链接。
  • 我们想要的是选择要在程序中的任何给定点调用的函数，以基于调用它的对象类型。此类操作称为动态链接或后期绑定。

纯虚函数

  • 基类中包含虚函数，以便在派生类中重新定义它以适应该类的对象，但是无法为基类中的函数提供有意义的定义。
  • = 0 告诉编译器该函数没有主体，称为纯虚函数。

C++接口

  • C++接口是使用抽象类实现的，这些抽象类不应与数据抽象混淆，数据抽象是将实现细节与关联数据分开的概念。

  • 一个类是通过将其至少一个函数声明为纯虚函数来抽象的。纯虚函数是通过在其声明中放置 “= 0” 来指定的

  • 抽象类（通常称为 ABC）的目的是提供一个适当的基类，其他类可以从中继承。抽象类不能用于实例化对象，只能用作接口。尝试实例化抽象类的对象会导致编译错误。

  • 因此，如果需要实例化 ABC 的子类，它必须实现每个虚函数，这意味着它支持 ABC 声明的接口。未能重写派生类中的纯虚函数，然后尝试实例化该类的对象，是编译错误。

  • 可用于实例化对象的类称为具体类。

STL

迭代器

  • 迭代器不是指针，是类模板，表现的像指针。他只是模拟了指针的一些功能，重载了指针的一些操作
    符，->、、++、–等。迭代器封装了指针，是一个“可遍历STL（ Standard Template Library）容器内
    全部或部分元素”的对象，本质是封装了原生指针，是指针概念的一种提升（lift），提供了比指针更高
    级的行为，相当于一种智能指针，他可以根据不同类型的数据结构来实现不同的++，–等操作。
    迭代器返回的是对象引用而不是对象的值，所以cout只能输出迭代器使用    取值后的值而不能直接输出
    其自身。
  • list迭代器实现

C和指针

a++ 和 ++a

  • a+++++b

    等价于（（a++）++）+b，但是编译失败，因为（（a++）++）+b中先执行a++返回的结果不能作为左值，因此编译失败。

  • a++返回临时对象，在内存中无明确的存储地址，不能作为左值，

  • ++a可以作为左值

const修饰的指针

  • 一旦p指向一块内存空间，p指向的内存空间的值无法通过*p进行修改
  • 可以修改p指向的地址

int* const

  • 不能修改指针指向的地址；可以修改指针指向的值

隐式类型转换

  • 整型提升：将char类型b,c，转换为普通类型b,c，求和后进行截断

指针

  • 左值->一个位置

  • 右值->一个数值

  • *、++优先级相同，右结合性

指针运算

  • 标准允许指向最后一个元素的指针和指向其后面的内存位置指针进行比较；不允许指向第一个和指向它前面内存的指针比较

数组与指针

  • 数组与指针并不是完全相等的

  • 声明数组会为数组保留内存空间，接着创建数组名，这是一个常量

  • 声明一个指针变量，编译器只为指针本身保留内存空间

  • 前者初始化一个字符数组的元素，后者是一个真正的字符串常量；
  • 指针变量被初始化为指向这个常量的存储位置

可变参数列表

  • stdarg宏
  • 可变参数必须从头到尾按顺序访问

连接

  • 假定一个程序在一个源文件中包含了声明: long fop;而在另一个源文件中包含了 extern short foo;
  • 如果给long类型的foo赋了一个较小的值37，short类型的foo同时获得一个值37.那么我们知道运行改程序的硬件是一个低位优先的机器。
  • 如果给long类型的foo赋了一个较小的值37，short类型的foo同时获得一个值0.那么我们知道运行改程序的硬件是一个高位优先的机器。

字符串

查找

  • 在一个字符串中查找一个特定的字符
    • strchr查找第一个匹配的位置
    • strrchr查找最后一个匹配的位置
  • 查找一组字符中任意一个出现的第一个位置
  • 查找一个子串
  • 查找一个字符串前缀
    • group指定一个或多个字符，返回str起始部分匹配的任意字符的字符数
    • group指定一个或多个字符，返回str起始部分不与group匹配的任意字符的字符数
  • 查找标记：隔离各个单独的部分（token标记），丢弃分隔符
  • 错误信息：将OS设置的errno返回（全局的变量）
